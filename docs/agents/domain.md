# Domain Documentation

This project uses a single-context layout for domain documentation.

## Layout

- **Context:** `CONTEXT.md` at the repo root defines the domain language and core concepts.
- **Decisions:** `docs/adr/` contains Architectural Decision Records (ADRs).

## Consumer Rules

1.  **Read `CONTEXT.md` first** to understand the "ubiquitous language" of the project.
2.  **Check `docs/adr/`** before proposing major structural changes.
3.  **Update `CONTEXT.md`** when new domain concepts are introduced.
